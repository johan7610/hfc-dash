## ⚠️ Remaining work (NOT done in this pass — too risky for blanket sweep)

### P1 — Per-page judgement needed (~100+ files)

1. **Raw `bg-blue-600` / `bg-green-600` / `bg-red-600` on buttons** (33 instances across 23 files). Spec §3.5 says use `corex-btn-primary` / `corex-btn-danger`. Common in: `deals-v2/show.blade.php`, `docuperfect/signatures/*`, `compliance/fica/show.blade.php`, `admin/fault-reports/show.blade.php`. Fix per-button — keeping spacing/sizing.

2. **`onmouseover="..."` inline JS for row hover** (181 instances across 73 files). Spec §5.11 marks transitional; proper fix is to ensure `.ds-table tbody tr:hover { background: var(--surface-2); }` (already in CSS) covers them, then strip the inline handlers. Many are NOT on `.ds-table` though — they style other elements. Safe migration is per-element review.

3. **Mixed `layouts.corex` vs `layouts.corex-app`** — both layouts exist; spec §2.1 says `corex-app` is the default. Need to confirm whether `layouts.corex` should be deprecated or kept as a sibling. See ~70 files using `layouts.corex`. Recommend: investigation, not bulk-rename.

4. **`rounded-2xl` / `rounded-lg` on cards in `admin/dashboard.blade.php`** (and similar). Spec §1.8 standardises on `rounded-md`. Dashboard cards explicitly noted as a pending product decision in spec §9.10 — do not migrate without product sign-off.

5. **`<x-primary-button>` / `<x-secondary-button>` / `<x-danger-button>` usage** — only 8 files (`auth/*`, `profile/partials/*`). Per spec §4 Component Inventory, these are explicitly carved out as legacy auth-only. **No action needed** unless auth screens are rebranded.
